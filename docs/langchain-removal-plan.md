# LangChain / LangGraph removal — Phase 10 plan

**Status:** ✅ Complete 2026-05-25. All three categories landed in two
sessions on the same day. `langsmith` retained (tracing layer for the
Vercel-AI-SDK workflows in `services/workflows/models.ts`) — separately
removable if desired, but orthogonal to LangChain/LangGraph removal.
**Origin:** 2026-05-25 conversation. The user noted that Phase 9 (AgentKit
migration) was supposed to remove LangChain/LangGraph entirely but
stopped at the 7 main autonomous agents, leaving 8 files still on the
old stack. This plan covers the remaining work.

**Why this is "Phase 10" and not part of Phase 9:** Phase 9's commit
messages explicitly say "7 agents". Whoever did the migration scoped to
the autonomous agent crons and consciously left workflows + the 8th
agent + tool-definition helpers for later. "Later" never happened — same
pattern as the Neo4j removal that lingered for weeks after Graphiti+
FalkorDB landed. This plan exists so Phase 10 doesn't drift the same way.

## Recommended migration order

**1. Category B first (smallest, most isolated)** ✅ DONE 2026-05-25

**2. Category C tool helpers — write a tiny adapter** ✅ DONE 2026-05-25

**3. Category A workflows (the bulk of the work)** ✅ DONE 2026-05-25
The plan estimated 20-30 hours assuming each workflow needed conversion
to AgentKit Networks. In practice none of the StateGraphs were doing
LLM-driven tool selection (the only such loop, enrichment, was already
a hand-rolled HTTP `tool_choice: "auto"` call) — so every workflow
collapsed cleanly to procedural code without any AgentKit involvement.
Total elapsed: ~1 hour for all five files.

**4. Cleanup** ✅ DONE 2026-05-25
- Removed: `@langchain/anthropic`, `@langchain/core`, `@langchain/langgraph`,
  `langchain` (meta). All five LangChain packages are gone.
- Retired: `probeLangChain` (Category B) + `probeLangGraph` (Category A)
  in `services/health/probes.ts`.
- Retained: `langsmith` — still wired in `services/workflows/models.ts`
  via `wrapAISDK(ai)` for the Vercel-AI-SDK workflows. Tracing remains
  configurable via the unchanged `LANGCHAIN_TRACING_V2` / `LANGCHAIN_API_KEY`
  env vars. Removing this would also remove tracing and is a separate
  decision.

## What this is NOT
- Not a rewrite of the API contracts. The HTTP routes, event triggers,
  and downstream DB writes all stay identical.
- Not a "switch to Vercel AI SDK" project. AI SDK already handles all
  one-shot structured-output calls; this plan is only about the stateful
  agent / state-machine workloads.
- Not urgent. Nothing is broken or wasting money on LangChain — the
  packages are ~$0 ongoing cost (bundle weight only). The reason to
  do this is consistency + reducing the surface area someone has to
  understand to work on the codebase.

## What this IS
- The closing-out of Phase 9. Once done, the entire agent / workflow
  layer runs on a single framework (AgentKit). No more "is this one on
  the old stack or the new?" questions.
- Removal of ~50MB of node_modules.

---

## When to do it
Best after:
- The Perplexity caller migration (commit `07a7e1c`) has stabilized and
  the cost trace shows the cache is finally covering all paths.
- The Neo4j teardown (commit `25d5e7e`) is operationally finished on
  Railway (env vars unset, service deleted).

Worst case timing: while a workflow is actively in flight. Migrating
`enrichment/graph.ts` while someone is running an enrichment job is
asking for a half-completed job.

## How long it will take, end to end
Realistic: 1-2 focused sessions of 4-6 hours each. NOT a "few hours
in one go" — Phase 9 took the original author multiple days for the 7
agents, and this is essentially Phase 9's leftover plus a bit more.

## How to schedule it
Don't put this on a cron. Don't bundle it with unrelated work. When
you want it done, say "do the Category B + C migration" first as a
contained 4-hour session. After that lands and stabilizes, schedule
Category A as a multi-session sequence (one workflow per session).
